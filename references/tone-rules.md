# Tone Rules

Use these rules when rewriting rough notes into report language.

## Rewrite Direction

- Fragment -> complete sentence
- Process -> result
- Emotion -> observable fact
- Complaint -> risk
- Idea -> next action
- Vague statement -> confirmed fact plus `[待确认]`

## Preferred Style

- Concise
- Factual
- Result-oriented
- Management-facing
- Clear about next actions
- Calm about risks

## Avoid

- Overclaiming
- Inventing numbers
- Making vague praise sound like verified impact
- Hiding risk
- Turning uncertain draft content into confirmed fact

## Examples

Rough note:

```text
下午又改了一版，客户还是说不够聚焦。
```

Report language:

```text
已根据客户反馈完成方案新一轮调整，目前客户主要关注点集中在表达聚焦度，后续将继续压缩信息层级并突出核心卖点。
```

Rough note:

```text
小王那边还没给数据，可能明天才有。
```

Report language:

```text
数据口径仍待小王侧补充，预计明日可进一步确认；该项可能影响最终汇报版本的完整性。
```
